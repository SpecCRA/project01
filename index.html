Lesson 1

World Wide Web

The web is made up of HTML documents. HTML stands for hypertext markup language.

Components of the Web

Your computer and internet browser

The internet - the largest computer network

Servers - computers that host files that make up the web

Your computer makes a request to the internet through HTTP, hypertext transfer protocol, to servers. The servers send back corresponding HTML files via the internet to your broswer.

HTML

HTML documents are made up of text content (what you see) and markups (what it looks like).

HTML Tags and Elements

An element contains an opening tag, its contents, and a closing tag. This page shows a reference for HTML tags, what they do, and how they can be used.

Computers are literal beings.

Computers require specific and exact instruction in order to perform tasks. Any bit of ambiguity or error such as a typo can create huge problems later. Also never use internet explorer until Microsoft makes it awesome.

Remember the demo about making a peanut butter and jelly sandwich. Something as simple as open the bag of bread is easily understood by our human minds, but the computer requires more instruction to do it properly and gently. You don't want a torn up bag every time you go get bread out. That's just silly.

Inline vs Block

An inline element affects the line an element is in. A block element puts a box around the content. The box can be different sizes too.

Lesson 2

Learning how to learn

We all start at a level of ignorance where we don't know anything about a topic. We quickly move to awareness where we realize there is a skill that can be acquired. The ability develops slowly and will eventually become fluency within the topic with practice.

How do websites work?

A website is like a house where HTML files are the structure providing information such as where rooms, what kind of rooms, and how many rooms you have. CSS is your design. This may show what color everything is or what shameful posters you hang up. Javascript controls the interactive units of your website. This would be analogous to your garage door opener or shower.

Developer Tools: Chrome

Chrome has developer tools within the browser to show you the HTML and CSS of a website. It visualizes each part of code and highlights specific elements to their corresponding parts of the website. This also shows you the tree-like structure inside each block element. You can use this to see how others organize their code to do front end web development.

Boxes, Boxes, and More Boxes!

Websites can be split up into many boxes where each block corresponds to different parts of the website. You can boxify things down to a single element conceptually and reverse engineer what a certain block contains.

Lesson 3

CSS - Cascading Style Sheets

CSS is what changes the style of a website. This changes things such as font sizes, colors, positioning of blocks, and many others. The most specific rules are applied to every child element. A selector choose which element your styles will be applied too. This page is a good reference for CSS selectors and how they work.

Boxes, revisited

This image shows the components of a block. Each component can be changed accordingly to your own design and is dynamic. Your box sizes can either be fixed (some pixel size) or dynamic (a percentage of the page).

Positioning in CSS

This link provides a good starter guide to the flex box. It basically is what allows blocks to be next to one another. Normally, div blocks take up the entire line in a webpage. After you adjust for size, flex box overrides the positioning of your elements.

Don't forget to add different CSS style codes for the different browsers, or someone will send you very angry emails asking why things don't work

Organization

Code needs to be organized for you or other people to read it later on. Your future self will thank your past self later, preferably with some beer. Besides proper indentation, your code needs to be consistent. Images can be kept in one or more specific images folders.

The Process of Creating a Website

Code, test, and refine

  1. Stucture the page.
  2. Look for repeated styles and semantic elements
  3. Write your HTML
  4. Apply styles from broadest to narrowest
  5. Fix code by seeing what goes wrong

Remember that developer tools is a way to quickly adjust and visualize changes without consequence.

The last step is to validate your code to make sure it will display properly for all browsers. HTML validation is done here, and CSS validation is done here.
